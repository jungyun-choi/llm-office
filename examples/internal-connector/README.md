# Internal Connector 계약 예시

이 디렉터리는 사내 원문을 읽는 커넥터와 AI Office 제어면 사이의 최소 계약을 보여줍니다. 예시는 동작하는 비밀키나 실제 저장소 정보를 포함하지 않습니다.

- `job.request.json`: 제어면이 사내 커넥터에 전달하는 작업 지시
- `job.result.sanitized.json`: 하이브리드 모드에서 커넥터가 반환할 수 있는 정제된 결과

실제 구현에서는 커넥터가 제어면으로 outbound polling하거나 메시지 큐를 소비하게 합니다. 제어면이 사내망으로 직접 접속할 수 있게 inbound 포트를 여는 방식은 권장하지 않습니다.

커넥터는 요청을 받는 즉시 다음을 검증해야 합니다.

1. 계약 버전, 서명, nonce, 만료 시간
2. 요청한 작업이 에이전트의 허용 도구 범위 안인지
3. 원문 반출이나 Git 쓰기 같은 금지 플래그가 없는지
4. 실행 중 kill switch와 호출/시간 예산을 초과하지 않는지
5. 결과가 DLP와 사람 승인 정책을 통과했는지

`opaque_id`를 실제 사내 위치로 되돌리는 매핑은 커넥터 내부 evidence service에만 둡니다. 원문 위치를 추측할 수 있는 locator hash도 외부로 보내지 않습니다. 외부 제어면에는 저장소명, 경로, URL, 커밋 SHA, 심볼명, 원문 인용, 실제 성능 수치를 보내지 않습니다.
